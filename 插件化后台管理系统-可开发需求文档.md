# 插件化后台管理系统 - 可开发需求文档

## 1. 文档信息
- 文档版本：v1.0
- 创建日期：2026-05-09
- 适用范围：Go 后端 + Vue3 前端插件化后台管理系统
- 目标读者：产品经理、架构师、后端开发、前端开发、测试工程师、运维工程师

## 2. 项目背景与定位
构建一套企业级后台管理基座（Main Host），所有业务能力均以插件（Plugin）形式接入，主基座仅提供通用能力，不承载具体业务逻辑。

### 2.1 系统定位
- 可插拔：业务模块独立安装、启用、禁用、升级、卸载。
- 可扩展：支持未来新增任意业务插件，不影响基座稳定。
- 可维护：插件独立开发、测试、部署，降低耦合与回归风险。
- 可运营：通过管理界面在线管理插件生命周期，无需重启后端服务、无需重建前端主应用。

### 2.2 设计原则
- 主系统不写业务代码。
- 前后端解耦，插件契约化接入。
- 统一安全与权限入口，业务能力按插件扩展。
- 运行时动态注册路由与菜单。

## 3. 核心目标
- 后台功能全部插件化。
- 插件可在管理界面可视化管理。
- 业务迭代不影响主基座稳定性。
- 支持无限扩展的插件生态。

## 4. 总体架构设计

### 4.1 后端架构（Go）
技术栈：Go 1.21+、Gin、GORM、MySQL/PostgreSQL（任选其一）、Redis（会话/缓存可选）。

#### 4.1.1 主基座职责
- 认证与授权：登录、JWT 签发、Token 校验、统一 RBAC。
- 插件生命周期管理：安装、启用、禁用、升级、卸载。
- 插件注册中心：加载插件元数据、权限、菜单、后端路由定义。
- 动态路由分发：根据插件状态与权限进行路由可见性控制。
- 配置中心：插件级配置管理（加密存储敏感配置）。
- 审计与日志：插件管理行为审计。

#### 4.1.2 插件职责
- 作为独立 Go 模块开发和打包。
- 可提供自身 API 路由。
- 可定义并维护自身数据库表（通过迁移机制）。
- 可注册菜单、权限、前端路由元信息。
- 暴露标准化生命周期接口（Install/Enable/Disable/Upgrade/Uninstall）。

#### 4.1.3 动态加载策略（建议）
由于 Go 原生 plugin 在跨平台与版本兼容方面存在限制，建议采用“进程外插件 + 契约注册”的稳定方案：
- 插件后端服务独立进程（HTTP/gRPC）。
- 基座维护插件注册表及反向代理路由。
- 启用插件时动态挂载路由前缀（如 `/plugin/{pluginKey}/...`）。
- 禁用插件时立即摘除路由和菜单，不重启主服务。

该方案可满足“热插拔、不重启、动态加载”，同时避免 Go plugin 二进制兼容问题。

### 4.2 前端架构（Vue3）
技术栈：Vue3、Vite、Element Plus、Vue Router、Pinia。

#### 4.2.1 主基座职责
- 登录页、基础布局（侧边栏/头部/标签页）。
- 权限控制（按钮/路由级）。
- 插件管理页面（安装、启停、升级、卸载、配置）。
- 动态菜单与动态路由引擎。

#### 4.2.2 插件前端形态
- 每个插件独立打包（建议输出 remoteEntry 或 UMD/ESM 包）。
- 插件暴露页面组件与路由元数据。
- 主应用根据后端返回的路由信息动态加载远程组件。

#### 4.2.3 动态加载策略（建议）
- 方案 A：Module Federation（推荐，适配 Vite 生态插件）。
- 方案 B：远程 ESM + import() 动态加载。
- 插件启用后实时下发菜单与路由刷新。
- 插件禁用后卸载对应路由并回收菜单展示。

## 5. 插件统一规范（强制）

### 5.1 插件元信息
每个插件必须声明：
- 名称（name）
- 标识（key，唯一）
- 版本（version，SemVer）
- 描述（description）
- 图标（icon）
- 作者与联系方式（author，可选）

### 5.2 插件注册内容
- 权限列表（permissions）
- 菜单列表（menus）
- 前端路由定义（frontendRoutes）
- 后端 API 路由前缀（apiPrefix）
- 生命周期钩子实现

### 5.3 生命周期接口（后端）
```go
type PluginLifecycle interface {
    Install(ctx context.Context) error
    Enable(ctx context.Context) error
    Disable(ctx context.Context) error
    Upgrade(ctx context.Context, fromVersion, toVersion string) error
    Uninstall(ctx context.Context) error
}
```

### 5.4 插件状态机
- NOT_INSTALLED（未安装）
- INSTALLED（已安装）
- ENABLED（已启用）
- DISABLED（已禁用）
- UPGRADABLE（可升级）

状态约束：
- 未安装 -> 已安装 -> 已启用/已禁用
- 已启用 <-> 已禁用
- 已安装/已禁用/已启用 -> 可升级（检测到更高版本）
- 已禁用 -> 卸载（推荐限制：启用状态不可直接卸载）

## 7. 数据流与时序

### 7.1 菜单与路由加载流程
1. 前端登录成功，拿到 JWT。
2. 前端请求菜单接口 `/api/menus`。
3. 后端基于：用户角色 + 权限 + 插件启用状态，返回可见菜单树与路由元数据。
4. 前端动态注册路由。
5. 前端按路由元信息加载插件页面（本地组件或远程组件）。

### 7.2 插件启停实时生效流程
1. 管理员在插件管理页点击启用/禁用。
2. 后端执行生命周期钩子并更新插件状态。
3. 后端刷新插件注册中心和动态路由映射。
4. 前端轮询或 WebSocket/SSE 接收状态变化通知，刷新菜单与路由。
5. 整体过程无需重启服务、无需重构建前端。

## 8. 后端接口设计（可直接开发）

统一返回结构（建议）：
```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

### 8.1 插件管理接口

#### 8.1.1 获取插件列表
- Method：GET
- URL：`/api/plugin/list`
- Query：`status`（可选）, `keyword`（可选）
- Response.data：插件列表

#### 8.1.2 安装插件
- Method：POST
- URL：`/api/plugin/install`
- Body：
```json
{
  "packageUrl": "https://.../plugin.zip",
  "checksum": "sha256:..."
}
```

#### 8.1.3 启用插件
- Method：POST
- URL：`/api/plugin/enable`
- Body：
```json
{
  "pluginKey": "livekit"
}
```

#### 8.1.4 禁用插件
- Method：POST
- URL：`/api/plugin/disable`
- Body：
```json
{
  "pluginKey": "livekit"
}
```

#### 8.1.5 卸载插件
- Method：POST
- URL：`/api/plugin/uninstall`
- Body：
```json
{
  "pluginKey": "livekit"
}
```

#### 8.1.6 升级插件（建议补充）
- Method：POST
- URL：`/api/plugin/upgrade`
- Body：
```json
{
  "pluginKey": "livekit",
  "packageUrl": "https://.../livekit-v1.2.0.zip",
  "targetVersion": "1.2.0"
}
```

### 8.2 菜单接口

#### 8.2.1 获取当前用户菜单（含插件）
- Method：GET
- URL：`/api/menus`
- Header：`Authorization: Bearer <token>`
- Response.data：菜单树 + 前端路由元数据

## 10. 权限与安全要求
- 所有插件管理接口仅超级管理员可访问。
- 所有插件路由受 RBAC 控制。
- 插件安装包必须校验签名或校验和。
- 插件配置中的密钥类字段加密存储。
- 卸载插件前检查依赖关系（被依赖插件不可直接卸载）。
- 所有生命周期操作写入审计日志。

## 11. 非功能性要求
- 可用性：插件启停成功率 >= 99.9%。
- 性能：`/api/menus` 在 200 个菜单项下响应 < 300ms（P95）。
- 稳定性：单插件故障不影响主基座核心能力。
- 兼容性：插件需声明支持的基座版本范围。
- 可观测性：暴露插件运行状态、健康检查、错误率指标。

## 12. 前端页面与交互要求

### 12.1 插件管理页
- 列表字段：名称、标识、版本、状态、更新时间、操作。
- 操作按钮：安装、启用、禁用、升级、卸载、配置。
- 状态标签：未安装/已安装/已启用/已禁用/可升级。
- 支持关键字搜索、状态筛选、批量操作（可选）。

### 12.2 动态菜单与路由
- 首屏加载后按用户权限拉取菜单。
- 路由按后端元数据动态注册。
- 插件组件加载失败时显示降级页面和重试入口。

### 12.3 远程组件加载
- 支持超时、重试、fallback 机制。
- 组件加载失败不影响主框架页面可用。

## 14. 测试与验收标准

### 14.1 功能验收
- 可通过 UI 完成插件安装、启用、禁用、升级、卸载全流程。
- 插件启用后菜单与路由实时可见。
- 插件禁用后菜单与路由实时隐藏。
- 插件卸载后相关注册信息清理完整。

### 14.2 稳定性验收
- 插件异常不导致主服务崩溃。
- 连续 100 次启停操作无内存泄漏显著增长。

### 14.3 性能验收
- 插件数量 100+ 时，插件列表接口 P95 < 500ms。
- 菜单接口 P95 < 300ms。

### 14.4 安全验收
- 未授权用户无法调用插件管理接口。
- 插件安装包篡改可被识别并拒绝安装。

## 15. 风险与应对
- Go 原生 plugin 兼容性风险：采用进程外插件与契约注册模型。
- 插件质量参差风险：引入插件市场准入校验（签名、版本、健康检查）。
- 动态前端加载失败风险：增加 fallback 页面和熔断重试。
- 插件依赖冲突风险：建立依赖声明与卸载前校验机制。
