# 插件开发规范（后端 + 前端 + 数据库）

## 1. 目标
每个插件都是一个独立业务单元，至少包含：
- 后端模块（API、生命周期、权限、菜单注册）
- 前端模块（路由页面、远程入口）
- 数据库迁移（新增表、升级脚本）

## 2. 推荐目录结构

```text
plugins/
  livekit/
    backend/
      module.yaml
      internal/
      migrations/
        001_init.sql
        002_add_index.sql
    frontend/
      package.json
      src/
      dist/
        remoteEntry.js
```

## 3. 插件后端契约

### 3.1 module.yaml 建议字段

```yaml
name: LiveKit 管理
key: livekit
version: 1.0.0
description: 音视频房间与节点管理插件
apiPrefix: /plugin/livekit
frontendEntry: /plugins/livekit/remoteEntry.js
remoteModule: ./App
permissions:
  - livekit:view
  - livekit:manage
menus:
  - name: livekit Dashboard
    path: /plugins/livekit
    component: RemotePluginPage
    icon: Grid
```

### 3.2 生命周期接口
- Install: 安装时执行（创建表、初始化配置）
- Enable: 启用时执行（注册路由、开放菜单）
- Disable: 禁用时执行（摘除路由和菜单）
- Upgrade: 升级时执行（迁移脚本）
- Uninstall: 卸载时执行（可选保留数据）

## 4. 插件前端契约

### 4.1 远程入口
插件前端打包后需产出远程入口（例如 `remoteEntry.js`）。

### 4.2 页面导出
远程模块建议默认导出 Vue 组件：

```js
export default {
  name: 'LivekitPluginPage',
  template: '<div>LiveKit Plugin</div>'
}
```

### 4.3 路由元数据
后端菜单接口需返回：
- `component = RemotePluginPage`
- `frontendEntry`
- `remoteModule`
- `pluginKey`

主应用根据元数据动态加载远程页面。

## 5. 数据库迁移规范

### 5.1 文件命名
- `001_init.sql`
- `002_add_xxx.sql`
- `003_alter_xxx.sql`

### 5.2 执行策略
- Install 阶段：执行全部未执行迁移
- Upgrade 阶段：按版本执行增量迁移
- 迁移必须幂等（重复执行不会破坏数据）

### 5.3 表命名建议
- 使用插件前缀，例如：
  - `livekit_rooms`
  - `livekit_nodes`
  - `livekit_sessions`

## 6. 插件接入流程
1. 开发插件后端与前端
2. 编译前端并产出远程入口
3. 准备插件安装包（包含 module.yaml + 迁移脚本）
4. 管理后台执行安装
5. 启用插件后菜单自动出现

## 7. 当前基座已支持能力
- 插件安装/启用/禁用/升级/卸载
- 插件配置中心
- 动态菜单
- 远程组件加载入口
- SQLite/MySQL 插件元数据持久化
